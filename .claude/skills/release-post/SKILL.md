# Build-in-public release post

When a Tulasi milestone ships, write a LinkedIn post announcing it with this
shape:

- **120–180 words.** Not shorter (thin), not longer (nobody reads it).
- **Honest voice.** First person, plain language, no marketing-speak.
- **One real struggle.** Every post names something that was actually hard
  or that didn't work the first time — build-in-public content is only
  interesting if it's honest about the messy parts, not a highlight reel.
- **Never overclaim.** Describe exactly what works today. Don't imply
  features that aren't built yet, don't round up percentages, don't call an
  early prototype "done."
- **Max 6 hashtags**, always including `#Tulasi` and `#BuildInPublic`.

Structure that tends to work: what shipped → the one thing that fought back
→ what it unlocks next. Skip generic hype openers ("Excited to share...").
Start with the concrete thing.
